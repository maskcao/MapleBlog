# 贡献指南

感谢你考虑为 MapleBlog 做出贡献！

## 如何贡献
- Fork 仓库并新建特性分支。
- 本地运行：`npm install` 后执行 `npm run dev`。
- 遵循现有的代码风格与项目结构。
- 需要时补充文档或测试用例。
- 提交 PR 并在描述中说明变更动机与内容。

## 提交规范
- 使用清晰、简洁的提交信息。
- 变更尽可能小而聚焦，避免混合多个不相关改动。

## 开发环境
- Node.js 18+（推荐 20/22）
- 提交前执行 `npm run build` 确认生产构建通过。

## 报告问题
- 请提供运行环境、复现步骤、期望行为与截图（如果可用）。
